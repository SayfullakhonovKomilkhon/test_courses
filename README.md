# Тестовое задание — IT-курсы 📱

Мобильное Android-приложение для просмотра, сортировки и сохранения курсов в «Избранное». Реализовано на языке **Kotlin** с разделением на фичи-модули и использованием современных архитектурных подходов.

---

## 🚀 Основные фичи и выполненные требования ТЗ

### 🔐 1. Авторизация (`feature:auth`)
* **Валидация полей ввода**: 
  * В поле Email запрещен ввод кириллицы (реализован `InputFilter`, перехватывающий ввод на лету как с клавиатуры, так и при вставке).
  * Email валидируется по строгой маске `текст@текст.текст`.
* **Активность кнопки «Вход»**: Кнопка неактивна, если хотя бы одно поле не заполнено или Email не соответствует маске.
* **Переход по кнопкам соцсетей**: При клике на кнопки VK и Одноклассники осуществляется переход во внешний браузер на ссылки `https://vk.com/` и `https://ok.ru/`.
* **Упрощенное тестирование (Offline Fallback)**: При возникновении сетевых ошибок авторизация завершается успешным входом, позволяя протестировать приложение под любыми валидными учетными данными.

### 📚 2. Каталог курсов (`app` & `core:network`)
* **Сетевые данные**: Загрузка списка курсов производится напрямую с Google Drive API по ссылке из ТЗ. Разбор JSON-ответа обернут в структуру `CoursesResponse` для парсинга Moshi.
* **Верстка карточек**:
  * Реальные баннеры для курсов по *Java*, *Python* и *3D-дизайну* загружены в ресурсы и отображаются на карточках.
  * Высота картинок приведена к значению **`114dp`** согласно макету (`328 × 114`).
  * Для картинок настроено скругление углов **`12dp`** со всех сторон с устранением белых артефактов на углах с помощью прозрачной подложки.
  * Описание курса строго ограничено 2 строками (`maxLines="2"`, `ellipsize="end"`).
* **Сортировка по дате публикации**: При нажатии кнопки сортировки список курсов перестраивается по убыванию даты публикации (`publishDate`). Кнопка интерактивно меняет цвет текста на зеленый (`#13C268`) и обновляет текст для наглядности.

### ❤️ 3. Избранное (`core:database` & Room)
* **Локальное хранение**: Добавление курса в избранное записывает его в базу данных Room (`favorite_courses`). Раздел «Избранное» работает реактивно на `Flow`.
* **Зеленая заливка**: Если курс в избранном, флаг-закладка на карточке заливается зеленым цветом (`#13C268`), иначе отображается белый контур.
* **Первоначальный импорт (Initial Sync)**: При самом первом запуске приложения, если БД пуста, курсы с флагом `"hasLike": true` из JSON-ответа автоматически сохраняются в локальную базу данных.

### 🏁 4. Онбординг (`feature:onboarding`)
* **Красивый баннер**: Старый текстовый staggered-список категорий заменен на цельное высококачественное изображение тегов согласно макету.
* **Адаптивный интерфейс**: Добавлен расчет системных отступов `ViewCompat.setOnApplyWindowInsetsListener`. Кнопка «Продолжить» автоматически сдвигается вверх на высоту системного меню навигации Android, предотвращая перекрытие.

## 🛠 Технологический стек и архитектура
* **Язык**: Kotlin
* **Архитектурный шаблон**: Clean Architecture + MVVM + Feature Modules
* **Внедрение зависимостей**: Koin
* **База данных**: Room (SQLite)
* **Сетевой клиент**: Retrofit + Moshi (JSON parser)
* **Адаптер списков**: AdapterDelegates (чистая и модульная работа с RecyclerView)

## ⚙️ Инструкция по запуску
1. Убедитесь, что у вас установлена **Android Studio** (последняя версия).
2. Склонируйте проект и откройте его в Android Studio.
3. Проект адаптирован под совместимость с Gradle `9.1.0` и Android Gradle Plugin (AGP) `9.0.0` для предотвращения ошибок импорта на старых версиях Studio.
4. Создайте файл `.env` в корневом каталоге проекта и укажите ключ Gemini API:
   ```env
   GEMINI_API_KEY=your_key_here
   ```
5. Запустите приложение на эмуляторе или физическом устройстве.
